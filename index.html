diagram type declaration

sequence — arrows and messages

participant Alice as Alice (user)    // declare with display alias
actor Bob                            // actor renders as a person shape

Alice->>Bob: sync call               // solid line, filled arrowhead
Bob-->>Alice: reply                  // dashed line (response)
Alice-)Bob: fire and forget          // open async arrow
Bob-xAlice: timeout / error          // X at receiver
Alice->>+Bob: call (activate)        // + activates lifebar
Bob-->>-Alice: reply (deactivate)    // - deactivates lifebar

sequence — create and destroy participants

create participant Session           // participant appears mid-diagram
Alice->>Session: init(userId)
create actor Auditor                // create an actor mid-flow
Session->>Auditor: write entry
destroy Session                     // destroy then a message FROM that participant
Session-->>Alice: token expired     // X mark drawn on Session lifeline here
destroy Auditor
Auditor-->>Alice: log flushed       // X mark drawn on Auditor lifeline here
create and destroy work exactly as on sequencediagram.org — participants can appear and disappear mid-flow
destroy draws an X through the lifeline at that point

sequence — autonumber and jump

%% #5                               // jump: next arrow at this level becomes 5 (or e inside a block)

// example — skip step 4 on the fast path:
A->>B: step one                     // 1
A->>B: step two                     // 2
A->>B: step three                   // 3
alt fast path (skip step 4)
    %% #5                           // jump to e (5th letter) inside this block
    A->>B: step five                // 3e
else normal path
    A->>B: step four                // 3d  ← use %% #4 before to land exactly on d
end
A->>B: step six                     // 4 (top-level continues)
top level uses numbers (1 2 3), inside any block uses letters (a b c)
%% #N on its own line jumps the counter — use inside alt/else to skip steps conditionally
the "from N" input in the toolbar shifts the entire top-level sequence start

sequence — blocks and grouping

loop Retry up to 3 times            // repeat block
    Alice->>Bob: ping
end

alt success                         // conditional — first branch
    Bob-->>Alice: 200 OK
else failure
    Bob-->>Alice: 500 Error
end

opt if feature flag is on           // optional block
    Alice->>Feature: check
end

par thread 1                        // parallel flows
    Alice->>Log: write
and thread 2
    Alice->>Cache: warm
end

break on circuit breaker open       // early exit
    Alice-->>User: 503
end

critical acquire lock               // mutual exclusion
    Alice->>DB: update
option timeout
    Alice-->>User: retry
end

sequence — notes, partitions and refs

note over Alice,Bob: TLS 1.3        // note spanning participants (mustard)
note right of Bob: rate limited     // note on one side
note left of Alice: initiator

rect rgb(220, 220, 240)             // shaded partition region
    Alice->>Bob: scoped calls
end

ref over Alice,Bob: Authentication flow   // reference to another diagram
forward arrows render black · backward arrows render teal
alt / opt / break / critical blocks highlight light-red

state — transitions

stateDiagram-v2
    [*] --> Idle             // initial transition
    Idle --> Processing : submit
    Processing --> Done : success
    Processing --> Error : failure
    Error --> Idle : retry
    Done --> [*]             // final state

    state Processing {       // composite (nested) state
        [*] --> Validating
        Validating --> Executing
        Executing --> [*]
    }

    state Choice <<choice>>  // choice pseudostate
    Idle --> Choice
    Choice --> Fast : score > 90
    Choice --> Slow : score <= 90

    Sending --> Sending : keepalive   // self-transition
    note right of Error: triggers alert

gantt — tasks and milestones

gantt
    title sprint 12 plan
    dateFormat YYYY-MM-DD
    excludes weekends

    section discovery
        requirements    : done,    t1, 2024-01-02, 3d
        spike           : active,  t2, after t1,   2d

    section build
        api endpoint    : crit,    t3, after t2,   5d
        frontend        :          t4, after t2,   7d
        integration     : crit,    t5, after t3,   3d

    section release
        go-live         : milestone, m1, after t5, 0d
        hypercare       :            t6, after m1, 5d
status values: done · active · crit · milestone (or leave blank)
after t1 = starts when task t1 ends · duration: 5d / 2w / 1h

mindmap — hierarchy and node shapes

mindmap
    root((Central Topic))   // circle
      Branch A
        Leaf 1
        Leaf 2
      Branch B[Square node]  // square brackets = rectangle
        Leaf 3
          Deep leaf
      Branch C(Rounded)      // parentheses = rounded
      Branch D{{Hexagon}}    // hexagon
indentation defines the hierarchy — each extra indent level = one level deeper
node shape is set by the brackets around the text: (( )) · [ ] · ( ) · {{ }}

validation rules (architecture mode only)

everything must be lowercase — uppercase triggers an error on that line
component names must be in square brackets: device [my router]
spelling sensitive — typos show suggestions: "reed" is not a valid colour. try: red
comments start with // and are ignored by the parser
uml and mermaid modes skip archsketch validation — native mermaid rules apply
nothing ever leaves your machine · no login · no storage · no telemetry
